Read External Storage

GCA Launcher requires access to your devices external memory for an assortment of reasons. The Wallpaper API will require permission due to the colors being processed by the bitmap it extracts and processed on your device memory using cache. The conversion process cannot take place unless it has a home for the file being processed.

GCA Launcher will also store user files and launcher database cache files on your device to help process actions and read data faster. Files include user data cache, saved shared preferences, crash logs.

Internet Acess

This is needed to send and receive ota messages to users and to receive issue reports as needed.

Access Notification Policy and Expand Statusbar

This is needed to expand the statusbar from the launcher interface. GCA Launcher also captures notifications for the priority preview feature and application icon badges.

In-App Purchases and Billing

This is needed to purchase Launcher Pledges to support development. This is a form of non-obligated donation system to support funding for the project. Last years pledges funded V2 development.

Request Delete Packages

This is needed to allow the user to delete apps from the launcher using app actions.

Bind AppWidget

This is needed to allow GCA to use application widgets in the desktop interface.

Package Usage Stats

This is needed to allow the user to use the predictive apps feature.
